What actually drives the price

The common misconception is that "more pages = more money." It's barely a factor. The real cost drivers are:

1. Custom design vs template

A site built on a Webflow or WordPress template can ship for $500. A pixel-custom design with bespoke motion, custom illustrations, and an original visual system starts at $2,000 and goes up fast. Design is the single biggest line item on most quotes.

2. Backend complexity

Static informational sites with no logins, no database, no payments — cheap. The moment you add user accounts, payments, dashboards, admin panels, file uploads, or third-party integrations, you've doubled the work. A login form is not "just a login form" — it's password reset, email verification, session management, rate limiting, and security review.

3. Content readiness

If you hand me final copy, final brand assets, and approved visual direction on day one, the project finishes in half the time. If I have to write your copy, source stock images, and run three design rounds — that's a different invoice.

4. Integrations

Each third-party integration (Stripe, Calendly, HubSpot, Mailchimp, Zoho, etc.) adds 2–8 hours of work depending on complexity. Five integrations stacks up fast.

5. Performance and SEO budget

"Make it fast" sounds simple. Hitting 90+ Lighthouse scores across all four categories (performance, accessibility, best practices, SEO), plus sitemap, structured data, OG cards, and a noscript fallback — that's a real chunk of work. Worth it for sites that need traffic; overkill for an internal portal.

What's NOT worth paying extra for

  • Custom CMS. Use Sanity, Contentful, or just Markdown files. Building one from scratch is a vanity tax.
  • "Mobile app" version of your site. A responsive web app is almost always enough. Skip the React Native rewrite until you have real demand.
  • "AI features" added late in the project. Either bake them in from day one or ignore them. Bolted-on AI features are the #1 reason scope creeps.
  • Animations everywhere. Pick three places where motion adds meaning. Animating everything is amateur.

Recurring costs nobody warns you about

Beyond the build cost, every site has ongoing expenses:

  • Domain: $10–20/year
  • Hosting: $0–25/month for static sites; $5–80/month for apps; $50–500/month for SaaS at scale
  • Email service (Postmark, SendGrid): $10–30/month if you send transactional email
  • Analytics: $0 (GA4) to $19+/month (Plausible/Fathom)
  • Maintenance: most freelancers offer retainers at $200–800/month for small fixes, content updates, and patches

How I price my own projects

I quote a fixed fee per project after a free 30-minute scoping call. The scope and timeline are written down before any code is written. If new work shows up mid-build, we agree on a change order — no silent scope creep, no surprise invoices. The fee covers production deployment, full handover docs, and 30 days of post-launch support.

Hourly billing punishes the developer for being fast and punishes the client for asking questions. Fixed fees align incentives.

What you should ask before signing

  1. "Can I see three similar projects you've finished and shipped to production?"
  2. "Who owns the code at the end?" (Answer should always be: you do.)
  3. "What's included in post-launch support, and what's not?"
  4. "What happens if you disappear mid-project?"
  5. "Will I get the source code, or only access to a hosted dashboard?"
